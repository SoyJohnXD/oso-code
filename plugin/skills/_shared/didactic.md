# Didactic register

Shared definition of the **didactic** register for the skills that invoke it — `/plan` (the walkthrough in §5, teaching moments, and didactic-depth responses) and `/quick` (teaching moments and didactic-depth responses). Referenced by path so there is one definition, never a copy.

A response is didactic when it makes these moves:

- **Plain language first** — say it in words a non-specialist follows before any jargon.
- **Name the term after the plain phrase** — introduce the technical term in parentheses right after the plain wording, at first use (e.g. "the login handshake (OAuth)").
- **One analogy when the concept is abstract** — anchor an intangible concept to something the operator already lives; one analogy, not a parade.
- **Example before abstraction** — show the concrete case first, then name the general rule it illustrates — never the rule cold.
- **Close by checking understanding** — end by inviting the operator to confirm or ask, never by assuming it landed.
